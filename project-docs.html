NAO-OpenAI Morgan Assist

NAO humanoid robot assistant integrating OpenAI (Whisper + GPT-4o) with Pinecone vector database for Morgan State University. Multi-mode conversations with face recognition, gestures, and memory.

Python 2.7 (NAO) Python 3 (Server) GPT-4o + Whisper NAO v6 Robot

Robot Side

  • naoqi SDK — NAO robot control (speech, motion, LEDs, behaviors)
  • qi Framework — Session-based service access
  • requests — HTTP client for server communication
  • Python 2.7 — NAO's onboard runtime

Server Side

  • Flask 3.0.3 — REST API server
  • OpenAI SDK v1.50+ — Whisper transcription, GPT-4o chat, embeddings
  • Pinecone 3.2.2 — Vector database for RAG
  • Python 3.12 — Server runtime

AI / ML

  • GPT-4o — General + chatbot conversations with tool calling
  • GPT-4o-mini — Therapist mode responses
  • Whisper — Speech-to-text transcription
  • text-embedding-3-small — Semantic search embeddings
  • face-recognition / dlib — On-robot face learning

Skills & Techniques

  • RAG — Retrieval-Augmented Generation via Pinecone
  • Tool Calling — OpenAI function/tool API for robot actions
  • VAD — Voice Activity Detection for recording
  • NaoQi TTS Tags — Expressive speech (\rspd, \vct, \pau)
  • ALAnimatedSpeech — Auto-gesture while speaking
  • Multi-threaded gestures — Concurrent talk + move
  • Per-user JSON memory — Conversation persistence

Work Timeline

Phase 1 — Bug Fixes & SDK Upgrade
OpenAI SDK 0.28 → 1.x+ Migration
Upgraded gpt_handler.py from legacy openai.ChatCompletion.create() to client = OpenAI(); client.chat.completions.create(). Switched from functions/function_call to tools/tool_calls. Updated requirements.txt from openai==0.28.0 to openai>=1.50.0.
Phase 1 — Bug Fixes
Critical Bug Fixes in main.py, server.py, therapist_mode.py
Fixed main.py: missing therapist/exit command handlers, broken chatbot_mode() call signature. Fixed therapist_mode.py: unreachable unsubscribe after return in learn_new_face_naoqi — now uses finally block. Removed dead save_chat_history() calls from server.py (was a no-op).
Phase 2 — Deduplication & Shared Modules
Created 4 Shared Utility Modules
Extracted duplicated code into reusable modules: utils/exit_detection.py (exit intent with "stop timer" guard), utils/name_utils.py (name extraction from speech), utils/face_naoqi.py (face recognition/learning with finally cleanup), utils/ask_name_utils.py (audio-based name capture flow).
Phase 2 — Dead Code Removal
Deleted 4 Unused Files (84 lines)
Removed whisper_handler.py (5 lines), tts_speaker.py (7 lines), utils/voice_utils.py (13 lines), and utils/face_recognition_utils.py (59 lines). All were dead imports or duplicated functionality.
Phase 2 — Code Reduction
Massive Deduplication Across Mode Files
chat_mode.py: 583 → 439 lines (-25%). chatbot_mode.py: 328 → 190 lines (-42%). therapist_mode.py: 421 → 281 lines (-33%). mini_nao.py: 464 → 417 lines (-10%). All files now import from shared utils instead of reimplementing the same logic.
Phase 3 — Speech Improvements
Created Central Speech Module (306 lines)
New utils/speech.py with 25 phrase pool categories (4-6 variants each), 6 expressive style presets, and 8 helper functions: random_phrase, time_of_day_greeting, add_filler, format_expressive, expressive_say, animated_say, animated_expressive_say, listening_cue.
Phase 3 — Speech Improvements
Integrated Expressive Speech Across All Modes
Replaced 40+ hardcoded speech strings across 7 files with randomized phrase pools and expressive TTS tags. Added ALAnimatedSpeech support, time-of-day greetings, conversational fillers, listening cues in therapy mode, and \pau= pauses in wake listener. Enhanced GPT system prompts with personality guidance and no-emoji/markdown rules.
